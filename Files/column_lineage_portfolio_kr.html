Snowflake 컬럼 계보 자동화 프로젝트

Snowflake 네이티브 LLM 기능과 Claude를 결합해 9,000개 이상의 객체를 대상으로 컬럼 단위 계보를 자동화한 작업입니다. 수개월이 걸릴 수 있는 수작업 문서화를 수주 단위로 줄이는 것이 핵심 목표였습니다.

Snowflake LLM Column Lineage Enterprise Data Warehouse

문제 정의

왜 이 작업이 필요했는가

엔터프라이즈 데이터웨어하우스가 커질수록 컬럼 수준의 데이터 흐름을 문서화하는 비용이 기하급수적으로 커집니다. 사람이 직접 계보를 추적하면 시간이 오래 걸리고, 유지보수 속도도 따라가지 못합니다.

접근 방식

모델을 직접 쓰는 대신, 구조를 먼저 설계

핵심은 단순히 LLM에 SQL을 던지는 것이 아니라, Snowflake 안에서 실행 가능한 흐름과 검증 기준을 같이 설계하는 것이었습니다. 모델 응답을 바로 믿지 않고, 사람이 확인할 수 있는 형태로 정리하고 보완하는 구조를 만들었습니다.

성과

문서화 시간을 수개월에서 수주로 단축

  • 9,000개 이상 객체를 대상으로 자동화 가능한 계보 추출 흐름을 설계했습니다.
  • 컬럼 단위 매핑에서 약 80% 수준의 정확도를 확보했습니다.
  • 실무 적용 가능성을 보여주면서 이후 확장 논의를 위한 기반을 만들었습니다.

의미

AI 실험이 아니라 운영 가능한 워크플로

이 프로젝트는 모델의 성능 자체보다, 엔터프라이즈 환경에서 믿고 사용할 수 있는 운영 구조를 어떻게 설계할지에 더 가까웠습니다. 제가 AI를 바라보는 방식도 이 지점에 있습니다.
